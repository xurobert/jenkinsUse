# 1. 需求概述

## 1.1 需求背景

基于REQ-2595：[《【REQ-2595】组织架构日常迭代V2.3》](https://alidocs.dingtalk.com/i/nodes/MNDoBb60VLaj1LZlhvA3O6z2JlemrZQ3?cid=382270710:1598198338&corpId=ding509fc72d6685d56d4ac5d6980864d335&doc_type=wiki_doc&iframeQuery=utm_medium=im_card&utm_source=im&utm_medium=im_card&utm_scene=person_space&utm_source=im)中对人员在企业中、项目中的状态机做了重新定义，因此本需求主要是针对状态机中的定义评估对数据指标的影响。

## 1.2 现状描述

基于基础方案：人员在项目中状态机的调整，不涉及组织在项目中的调整，因此班组的履约状态无变化。

![429ac0578601e15f313fa413fba50196.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/4jKqmVjpg1VvOw19/img/d00a1c12-b690-461a-9223-245396612535.png)![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/meonaAaP1rPjnXxj/img/ed1b955a-2301-448f-a3d0-fb29e42e97d4.png)

1.  当前在数据侧的定义中，暂时未使用到人员在企业中的状态。
    
2.  人员在项目中分为五个阶段：未审核、入场、在册、项目中、退场，其中：
    

1.  **未审核**：被邀请进入项目但是还处于审核中，尚未通过审审核；如：未审核工人数
    
2.  **入场**：人员审核通过；如：入场工人数
    
3.  **在册**：当前在项目中，且包含周期内退场；如：在册工人数、在册管理人员数等；
    
4.  **项目内**：当前在项目中，且包含历史退场；如：项目内工人数，项目内管理人员数，项目内班组长数；
    
5.  **退场**：已经退出项目； 如：退场工人数
